## 我已读取并将遵循的核心约束（来自两份文档）
- **视觉配色（必须落到 Tailwind 变量/Token）**：
  - Background：`#09090b`
  - Surface：`#18181b`
  - PrimaryAccent(HotPink)：`#ec4899`
  - SecondaryAccent(NeonGreen)：`#22c55e`
  - TextPrimary：`#fafafa`，TextMuted：`#a1a1aa`
  - 质感：1px 半透明粉/绿边框 + Glow 发光阴影；可选代码雨/警示条纹点缀
- **布局结构**：
  - PC：左侧固定 `Sidebar`，顶部粉色 Logo，菜单（Dashboard/Chat/Puzzles），底部探员身份卡（绿色在线）
  - Mobile：底部 `TabBar`
- **页面要素**：
  - Dashboard：大字报欢迎语 “WELCOME BACK, MAYI”，统计面板（荧光绿进度条），终端区域滚动播放梅子碎碎念
  - Chat：用户气泡（深灰底+绿边框+右对齐），梅子气泡（黑底+粉边框+左对齐）；输入框下划线聚焦粉色；发送按钮绿色回车符
  - Puzzles：卡片瀑布流/网格；hover 粉色边框 + glitch/毛玻璃扭曲；难度标签（绿/粉/红）
- **梅子人设红线（会直接影响 UI 文案与后续 Chat 文本样式）**：
  - 禁用八股开场与结构词：如“首先/其次/最后/总而言之”
  - 禁在回复开头用“好的/没问题/我理解”
  - 禁承认自己是虚拟助手/语言模型；必须咬死“桂电大一新生”身份
  - 对社会派推理保持不屑调性

## 阶段 1：项目初始化与全局配置（依赖安装、Tailwind 颜色变量配置）
【自我查验】：我必须随时回顾《UI 架构蓝图》中的色彩规范，确保使用了正确的黑/粉/绿配色，并回顾《灵魂设定》确保文案符合梅子的人设。
- **目标**：搭建 Next.js(App Router) 工程骨架、Tailwind 主题 Token、全局字体/背景/发光效果基础；为后续 3 个页面做可复用 UI 基础组件。
- **初始化与依赖（计划执行时落地）**：
  - Next.js(App Router) + TypeScript
  - Tailwind CSS（含 `@tailwindcss/typography` 可选，用于后续 Markdown）
  - `lucide-react` 图标
  - 可选：`clsx` + `tailwind-merge`（统一 class 合并）
  - 可选：`framer-motion`（轻量动效/hover 发光/弹窗出现）
- **需要创建/调整的文件**：
  - `package.json`（安装依赖）
  - `next.config.(js|mjs)`（如需要：允许本地静态资源/图片）
  - `postcss.config.js`、`tailwind.config.ts`、`app/globals.css`
  - `app/layout.tsx`（挂载全局样式/字体/背景纹理层）
  - `lib/cn.ts`（可选：className 合并工具）
  - `lib/theme.ts`（可选：集中管理颜色 token/阴影 token，避免散落 magic values）
- **Tailwind 主题与 CSS 变量落地方案**：
  - 在 `app/globals.css` 定义 CSS 变量（背景/表面/粉/绿/文本），并提供透明度衍生变量（用于 1px 边框）
  - 在 `tailwind.config.ts` 的 `theme.extend.colors` 映射到 `rgb(var(--...))` 或 `hsl(var(--...))`（二选一，优先 rgb 直观）
  - 统一 token 命名（建议）：
    - `bg.app`、`bg.surface`
    - `accent.pink`、`accent.green`
    - `text.primary`、`text.muted`
    - `border.pink`、`border.green`
  - 扩展 `boxShadow`：`glowPink`、`glowGreen`，扩展 `backgroundImage`：`matrixRain` / `cautionStripe`（可选，先做占位）
- **基础 UI 组件（先建壳，后续阶段直接复用）**：
  - `components/ui/GlowContainer.tsx`（统一容器：surface 背景、1px 半透明边框、hover 发光）
  - `components/ui/Button.tsx`（粉色主按钮、绿色次按钮、禁用态）
  - `components/ui/Badge.tsx`（难度标签：green/pink/red）
  - `components/ui/Progress.tsx`（荧光绿进度条）
  - `components/ui/Terminal.tsx`（终端框样式 + 绿色字体滚动区域，先静态）
  - `components/ui/Avatar.tsx`（梅子像素风占位：先图标/渐变圆）
- **验收标准（阶段 1 结束前）**：
  - 打开任意页面背景为 `#09090b`，容器层级为 `#18181b`
  - 粉/绿发光阴影与 1px 半透明边框可用
  - 组件库文件结构稳定，后续只“填内容”不“改地基”

## 风险点与默认取舍（不阻塞执行）
- **瀑布流实现**：默认先用“响应式网格 + 不同卡片高度”模拟；如你强制要 Masonry 真瀑布流，再引入轻量 Masonry 方案。
- **字体**：默认先用系统字体（标题加粗、tracking 拉开）快速对齐风格；你若指定字体（如 Impact 或特粗黑体替代），阶段 1 再落地。
- **shadcn/ui**：默认不引入，等需要复杂弹窗/下拉再加，避免早期依赖膨胀。
